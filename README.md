# SQL Practical Exercises

This repository contains a SQL practice project for a fictional bakery/order management system. It includes database creation scripts and a set of SQL challenge questions designed to build skills from beginner queries to advanced analytics and database design.

## Project Structure

- `DB_Createtion.sql` - Creates the `management_system` database and populates tables such as `Categories`, `Customers`, `Ingredients`, `Products`, `Orders`, `Recipes`, and `OrderDetails`.
- `Challenges.md` - Lists progressive SQL challenges from basic selection and filtering to advanced analytics, window functions, CTEs, data modeling, and integrity checks.

## Key Concepts Covered

- SELECT, WHERE, JOIN
- GROUP BY, HAVING, aggregates
- Subqueries and CTEs
- Window functions (`ROW_NUMBER`, `RANK`, `LEAD`, `LAG`)
- Data manipulation (INSERT, UPDATE, DELETE)
- Schema design, constraints, and indexing
- Data integrity and auditing
- Business reporting and forecasting queries

## How to Use

1. Open `DB_Createtion.sql` in your SQL editor.
2. Execute the script to create the database and sample data.
3. Review the challenge list in `Challenges.md`.
4. Write and run queries against the created schema to solve the prompts.

## Recommended Workflow

1. Start with Phase 1 questions to get comfortable with table structure and basic queries.
2. Move through Phase 2 and Phase 3 for joins, aggregation, and subqueries.
3. Use Phase 4 and later phases to practice CTEs, window functions, and more advanced reporting.
4. Keep solutions organized in a separate SQL file or notebook for review.

## Notes

- The schema uses sample bakery and café data with products, customers, orders, and ingredients.
- The challenge list is designed to scale from beginner to expert levels.
